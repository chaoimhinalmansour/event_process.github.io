Event Production System / 主轴 v3.0

校园活动流程

这不是一张活动安排表，而是一条可冻结、可交接、可验收、可降级、可复盘的校园活动生产线。业务模块可以替换，主轴必须完整跑完。

T-21 / T-14立项：确定目标、人群、主题、风格、预算、人数上限。
T-7方案冻结：主流程、物料规格、场地规则、邀约口径定稿。
T-3执行冻结：物料、人员、动线、Kit、应急药箱基本到位。
T-24h名单锁定：最终来访表确认，只允许例外补登。
D+1 / D+2跟进复盘：分层触达，沉淀下次活动修正项。

记录模板字段

后续可以拆成可打印表格或在线表单
表单 核心字段 用途 负责人
内部报名确认表 家庭编号、学生姓名、年级、家长姓名、电话、到访人数、是否开车、车牌号、关注方向、负责老师、确认状态、备注 锁定名单、安保放行、签到准备、咨询分配 董老师
物料规格与制作表 物料名称、尺寸/规格、数量、用途/点位、负责人、到位状态、验收人 保证物料不是临时想起来，而是在活动开始前统一到位 董老师主责；林老师配合制作；骆老师协助
活动 Kit 清点表 物品名称、标准数量、现有数量、缺口、补货人、补货时间、归还状态 让签字笔、胶带、剪刀、插线板等现场基础物资常备化 董老师主责；骆老师协助
应急药箱检查表 物品名称、数量、有效期、检查人、缺口、补充状态 只用于现场轻微突发情况的基础应急，不替代专业医疗处理 董老师清点；现场按应急情况处理
模块卡 / 模块冻结表 模块名称、目标家庭、核心问题、活动形式、标准时长、人数上限、场地设备、模块老师、物料、输出线索、咨询承接问题、降级方案 保证每场活动只替换模块，不改主轴；让模块结束后能自然进入咨询 模块老师填写；林老师确认方案；董老师确认承接
咨询记录表 家庭编号、学生姓名、咨询老师、主要问题、关注产品/路径、预算/时间节点、意向等级、下一步动作、预约时间、备注 承接转化、分层跟进、复盘转化效率 董老师
复盘报告 报名数、到访数、到访率、咨询数、A/B/C/D意向数、主要问题、现场风险、下次改进项、负责人、截止点 让下一场活动继承经验，而不是重新摸索 林老师整理；董老师、杨老师确认
定版原则：页面先帮助团队理解“怎么跑”，再帮助负责人检查“是否做到”。宣讲和业务模块后续作为子系统接入，不反向改变主轴。
